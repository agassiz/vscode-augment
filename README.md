## 📦 安装方法

### 本地 VSIX 安装包安装

**重要说明**：这是一个本地安装包，需要手动下载并安装，不能通过 VSCode 扩展市场直接安装。

#### 方法一：图形界面安装（推荐，适合一般用户）

1. 访问 [Releases 页面](https://github.com/agassiz/vscode-augment/releases)
2. 下载最新版本 `v0.524.1-25-0809` 的 `.vsix` 文件
3. 打开 VSCode，点击左侧扩展图标 (或按 `Ctrl+Shift+X` / `Cmd+Shift+X`)
4. 点击扩展面板右上角的 `...` (更多操作) 按钮
5. 选择 `从 VSIX 安装...` (Install from VSIX...)
6. 浏览并选择下载的 `.vsix` 文件
7. **重要**：安装完成后必须重启 VSCode

#### 方法二：命令面板安装

1. 下载 `.vsix` 文件后，在 VSCode 中按 `Ctrl+Shift+P` (Windows/Linux) 或 `Cmd+Shift+P` (macOS)
2. 输入 `Extensions: Install from VSIX...`
3. 选择下载的 `.vsix` 文件进行安装
4. **重要**：安装完成后必须重启 VSCode

#### 方法三：命令行安装（适合开发者）

```bash
# 下载 .vsix 文件后，在文件所在目录执行：
code --install-extension vscode-augment-0.524.1.vsix
```

#### 方法四：直接拖拽安装（最简单）

1. 下载 `.vsix` 文件到本地
2. 打开 VSCode 扩展面板 (`Ctrl+Shift+X` / `Cmd+Shift+X`)
3. 将 `.vsix` 文件直接拖拽到扩展面板中
4. 确认安装并重启 VSCode

> **💡 安装提示**：推荐使用方法一（图形界面）或方法四（拖拽），这两种方式对一般用户最友好，不需要使用命令行。

## 🚀 使用说明

**本地安装包特点**：
- 这是一个修改版的本地安装包，不依赖官方扩展市场

## 💡 重要使用说明

### 📧 邮箱要求
- **必须使用正常邮箱**：Gmail、Outlook、QQ邮箱等主流邮箱服务
- **禁止使用临时邮箱**：如 2925.com、10minutemail 等临时邮箱服务
- **风控提醒**：临时邮箱可能触发系统风控机制，导致功能异常
- **地区限制**：如遇到地区限制，请参考上方"地区限制解决方案"

### � 邮箱使用经验分享 - 重要提醒

> [!WARNING]
> **作者实际使用经验**：
>
> 我之前一直使用 **2925.com 临时邮箱**，用了好几个月，一直很好用，突然 **7月底（前几天）**，官方加强了风控，邮箱不好用了，开始以为是我的插件出问题了，后来使用 **自己的域名邮箱现在一直很稳定**；其它的像 **qq 邮箱我也没试过**；
>
> **⚠️ 重要建议**：如果您之前使用临时邮箱没问题，但最近突然失效，这很可能是 **官方风控升级** 导致的，建议立即切换到稳定的邮箱服务！

### ⚠️ 防封重要提醒

> [!IMPORTANT]
> **插件稳定性说明**：
>
> 如果使用插件后**不到几分钟就被封号**，请注意检查以下问题：
> - **IP问题**：确保使用纯净的IP地址，避免使用被污染的代理IP
> - **邮箱问题**：避免使用临时邮箱注册账号
> - **对话环境**：对话时确保走纯净IP，不要使用不稳定的网络环境
>
> **经过测试，插件本身是稳定的**。如果出现秒封情况，通常是由于：
> 1. **注册时使用了临时邮箱**
> 2. **对话时使用了不纯净的IP**
> 3. **网络环境不稳定**
>
> **建议**：使用正规邮箱注册 + 稳定纯净的网络环境，可以大大降低被封风险。
